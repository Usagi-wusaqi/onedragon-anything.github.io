本页对应一条龙界面左侧导航 `「设置」-「脚本环境」` 中的配置项，主要说明截图方法、代码同步源、Python/Pip 下载源、网络代理和脚本快捷键。不熟悉左侧各入口用途时，可先查看[界面导航](../gui_guide.md#设置页面)。

## 基础

### 截图方法

`截图方法` 决定脚本如何获取游戏画面。识别、自动战斗和截图助手都依赖这里的截图方式。

默认使用 `自动` 即可。自动模式会按 `Print Window`、`BitBlt`、`MSS`、`PIL` 的顺序尝试，某一种截图方式失败时会继续尝试后面的方式。

常见选择：

- `Print Window`：后台模式需要使用的截图方式。开启[游戏设置](./setting_game.md#_1-2-后台模式-测试版)里的 `后台模式（测试版）` 时，需要配合这个选项。
- `BitBlt`：直接截取窗口区域。B 服登录界面如果出现识别不到登录框、账号记录等情况，优先切换到 `BitBlt`。
- `MSS` / `PIL`：作为兼容方案使用。当前面的截图方式不可用、截图黑屏或识别异常时，可以依次尝试。

切换截图方法后，脚本会重新初始化控制器。若已经在运行任务，建议先停止任务，再切换截图方法并重新开始。

### 调试模式

`调试模式` 正常使用时无需开启。排查识别、截图或运行流程问题时，开启后会输出更多调试信息。

### 复制截图到剪贴板

开启后，按下脚本截图按键时，会自动把截图复制到剪贴板，方便提交问题或临时查看。

## Git 相关

### 代码源

`代码源` 控制脚本同步代码时使用的远端。国内无法访问 GitHub 时，可以切换到 Gitee。

### 自动更新

`自动更新` 控制使用 exe 启动时是否自动检测并更新代码。

### 强制更新

`强制更新` 会在同步时将代码更新到最新，并覆盖本地代码改动。普通用户不改源码时可以开启；如果你正在开发或保留本地改动，请谨慎使用。

## Python 相关

### Python 下载源

`Python下载源` 控制运行环境下载 Python 时使用的镜像源。旁边的 `自动测速选择` 可以测试并选择当前最快的下载源。

### Pip 源

`Pip源` 控制安装 Python 依赖时使用的镜像源。旁边的 `自动测速选择` 可以测试并选择当前最快的 Pip 源。

## 网络相关

### 网络代理

`网络代理` 用于加速下载和访问网络资源。

- `无`：不配置代理
- `个人代理`：使用你自己填写的代理地址
- `GitHub 代理`：使用免费代理地址，主要用于加速工具和模型下载，不能加速代码同步

### 个人代理

选择个人代理后，在这里填写代理地址，例如 `http://127.0.0.1:8080`。

### 免费代理

选择 `GitHub 代理` 后，会显示 `免费代理` 地址输入框。开启 `自动获取免费代理地址` 后，脚本会尝试自动获取；获取失败时可以前往 `https://ghproxy.link/` 查看并手动更新。

## 脚本按键

### 开始运行

`开始运行` 用于开始、暂停或恢复某个应用。

### 停止运行

`停止运行` 用于停止正在运行的应用。停止后不能恢复，只能重新开始。

### 游戏截图

`游戏截图` 用于开发和提交问题。截图会自动对 UID 打码，并保存到 `.debug/images/` 文件夹。

### 调试按钮

`调试按钮` 用于开发调试，部分应用会监听这个按键执行调试逻辑。
